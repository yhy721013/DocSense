

# 武器装备知识谱系解析接口及字段详解
```
接口路径： /llm/weaponry
请求方法： POST
```

## 一、请求参数字段说明（任务提交阶段）

### 📌 LLMReq 根对象
| 字段 | 类型 | 必填 | 说明 | 示例值 |
|------|------|------|------|--------|
| **businessType** | String | 是 | 业务类型标识，固定值 | `"weaponry"` |
| **params** | Object | 是 | 武器装备解析参数容器 | - |

---

### 📌 LLMWeaponryReq (params对象)
| 字段 | 类型 | 必填 | 说明 | 示例值 |
|------|------|------|------|--------|
**status** | String | 是 | **状态**：0未解析 1解析中 2已解析 3解析失败 | `0未解析 1解析中 2已解析 3解析失败` |
| **architectureId** | Long | 是 | **知识谱系ID**：标识当前解析任务所属的知识体系 | `1` |
| **weaponryTemplateFieldList** | List | 是 | **待解析字段列表**：定义需要从文档中提取的字段结构 | `[ {...}, {...} ]` |
---

### 📌 LLMWeaponryTemplateField (字段定义对象)
| 字段 | 类型 | 必填 | 说明 | 请求时要求 | 示例值 |
|------|------|------|------|------------|--------|
| **templateClassifyId** | Long | 是 | **模板分类ID**：字段在知识谱系分类id | 不需要改动 | `1772442376645740` |
| **fieldName** | String | 是 | **字段名称**：业务字段中文名 | 不需要改动 | `"操场长度"` |
| **fieldType** | String | 是 | **字段类型**：`INPUT`=普通文本字段`TABLE`=二维表格字段 | 不需要改动 | `"INPUT"` / `"TABLE"` |
| **fieldDescription** | String | 否 | **字段说明**：指导大模型提取内容的提示语 | 不需要改动 | `"根据文档提取操场实际长度"` |
| **tableFieldList** | List | 条件 | **表格结构**：- 仅当`fieldType="TABLE"`时必填- 外层List=行，内层List=列 | 子字段需清空解析字段 | `[[{...},{...}], [...]]` |

> 💡 **关键规则**：  
> - 所有层级（含`tableFieldList`嵌套字段）的 `analyseData` 和 `analyseDataSource` **必须清空**  
> - 服务端校验：若检测到非空值，将返回 `INVALID_PARAM` 错误

## 二、回调结果字段说明（服务端返回阶段）

### 📌 回调根对象
| 字段 | 类型 | 说明 | 示例值 |
|------|------|------|--------|
| **businessType** | String | 业务类型标识（用于客户端路由） | `"weaponry"` |
| **data** | Object | 填充完整解析结果的参数对象 | `LLMWeaponryReq` |
| **msg** | String | 处理状态描述 | `"解析成功"` / `"3个字段未匹配"` |

---

### 📌 LLMWeaponryTemplateField (回调填充后)
| 字段 | 类型 | 说明 | 示例值 |
|------|------|------|--------|
| **templateClassifyId** | Long | 是 | **模板分类ID**：字段在知识谱系分类id | 不需要改动 | `1772442376645740` |
| **fieldName** | String | 是 | **字段名称**：业务字段中文名 | 不需要改动 | `"操场长度"` |
| **fieldType** | String | 是 | **字段类型**：`INPUT`=普通文本字段`TABLE`=二维表格字段 | 不需要改动 | `"INPUT"` / `"TABLE"` |
| **fieldDescription** | String | 否 | **字段说明**：指导大模型提取内容的提示语 | 不需要改动 | `"根据文档提取操场实际长度"` |
| **tableFieldList** | List | 条件 | **表格结构**：- 仅当`fieldType="TABLE"`时必填- 外层List=行，内层List=列 | [[{},{}],[{},{}]]
| **analyseData** | String | **提取结果值**：- 成功提取：具体文本值- 未匹配：空字符串 | `"10米"` / `""` |
| **analyseDataSource** | List | **溯源证据链**：- 至少1条记录（无来源时返回空内容对象）- 按置信度降序排列 | `[ {...}, {...} ]` |

---

### 📌 LLMWeaponryAnalyseDataSourceReq (数据来源明细)
| 字段 | 类型 | 说明 | 示例值 |
|------|------|------|--------|
| **content** | String | **原文片段**：文档中直接提取的原始内容 | `"操场实测长度10米"` |
| **source** | String | **来源定位**：文献/文件名称+位置标识 | `"《校园设施验收报告》P15"` |
| **time** | String | **信息时间**：内容产生时间（非解析时间） | `"2023-08-10"` |
| **translate** | String | **翻译内容**：- 外文文献：提供中文翻译- 中文文献：空字符串 | `"Translation: Playground length 10m"` / `""` |

## 典型场景字段解读

### 场景1：普通字段（INPUT）
```json
{
    "fieldDescription": "操场长度字段描述根据文档中的信息提取",
    "fieldName": "操场长度",
    "fieldType": "INPUT",
    "templateClassifyId": 1772442376645740,
    "analyseData": "解析数据操场长10米",
    "analyseDataSource": [
        {
            "content": "内容",
            "source": "这段文献，这里内容",
            "time": "1999-12-01",
            "translate": "翻译后的：这段文献，这里内容"
        }
    ]
}
```
> **字段意义**：`analyseData` 是结构化结果值，`analyseDataSource` 提供可验证的原文依据

### 场景2：表格字段（TABLE）
```json
"tableFieldList": [
  [ // 第1行
    { "fieldName": "姓名", "analyseData": "张三", ... },
    { "fieldName": "分数", "analyseData": "80", ... }
  ],
  [ // 第2行
    { "fieldName": "姓名", "analyseData": "李四", ... },
    { "fieldName": "分数", "analyseData": "66", ... }
  ]
]
```
> **字段意义**：  
> - 外层List = 表格行（2行数据）  
> - 内层List = 每行的列字段（姓名+分数）  
> - 每个单元格独立包含 `analyseData` 和溯源证据

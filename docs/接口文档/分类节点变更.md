

# 文档分类节点变更接口

```
接口路径： /llm/reassign
请求方法： POST
```

## 一、功能说明

当前端修改了某个文档所属的分类节点（`architectureId`）后，通过本接口将变更同步至后端，使后续的知识谱系解析等操作能基于正确的分类节点进行。

> 💡 **核心行为**：  
> 本接口为**同步接口**，处理完成后直接返回结果，不涉及后台异步任务、进度推送或回调。

---

## 二、请求参数字段说明

### 📌 根对象

| 字段 | 类型 | 必填 | 说明 | 示例值 |
|------|------|------|------|--------|
| **businessType** | String | 是 | 业务类型标识，固定值 | `"reassign"` |
| **params** | Object | 是 | 变更参数 | `{...}` |

---

### 📌 params 对象结构

| 字段 | 类型 | 必填 | 说明 | 示例值 |
|------|------|------|------|--------|
| **fileName** | String | 是 | 文件对象名（业务主键） | `"bded228dc94440519d87f97cfb6b520b.pdf"` |
| **oldArchitectureId** | Long | 是 | 变更前的分类节点 ID | `1768464916588441` |
| **newArchitectureId** | Long | 是 | 变更后的分类节点 ID | `1768464916588442` |

> ⚠️ **校验规则**：  
> - `fileName` 不能为空  
> - `oldArchitectureId` 与 `newArchitectureId` 不能为空  
> - `oldArchitectureId` 与 `newArchitectureId` 不能相同  

---

## 三、请求示例

```json
{
  "businessType": "reassign",
  "params": {
    "fileName": "bded228dc94440519d87f97cfb6b520b.pdf",
    "oldArchitectureId": 1768464916588441,
    "newArchitectureId": 1768464916588442
  }
}
```

---

## 四、响应参数说明

### 📌 受理成功响应（HTTP 200）

| 字段 | 类型 | 说明 | 示例值 |
|------|------|------|--------|
| **businessType** | String | 业务类型标识 | `"reassign"` |
| **message** | String | 处理结果概述 | `"变更成功"` |
| **data** | Object | 变更结果详情 | 见下方结构 |

### 📌 data 元素结构

| 字段 | 类型 | 说明 | 示例值 |
|------|------|------|--------|
| **fileName** | String | 文件对象名 | `"bded228dc94440519d87f97cfb6b520b.pdf"` |
| **oldArchitectureId** | Long | 变更前分类节点 ID | `1768464916588441` |
| **newArchitectureId** | Long | 变更后分类节点 ID | `1768464916588442` |
| **success** | Boolean | 本条记录是否变更成功 | `true` |
| **message** | String | 本条记录处理描述 | `"变更成功"` / `"文档记录不存在"` |

---

## 五、响应示例

### 变更成功

```json
{
  "businessType": "reassign",
  "message": "变更成功",
  "data": {
    "fileName": "bded228dc94440519d87f97cfb6b520b.pdf",
    "oldArchitectureId": 1768464916588441,
    "newArchitectureId": 1768464916588442,
    "success": true,
    "message": "变更成功"
  }
}
```

### 文档记录不存在

```json
{
  "businessType": "reassign",
  "message": "变更失败",
  "data": {
    "fileName": "bded228dc94440519d87f97cfb6b520b.pdf",
    "oldArchitectureId": 1768464916588441,
    "newArchitectureId": 1768464916588442,
    "success": false,
    "message": "文档记录不存在"
  }
}
```

---

## 六、错误响应

### 参数校验失败（HTTP 400）

```json
{
  "error": "fileName不能为空"
}
```

```json
{
  "error": "oldArchitectureId与newArchitectureId不能相同"
}
```

```json
{
  "error": "params不能为空"
}
```

```json
{
  "error": "businessType必须为reassign"
}
```

---

## 七、处理逻辑说明

1. **参数校验**：校验 `businessType`、`params` 非空、各必填字段存在且合法。
2. **文档记录查询**：根据 `fileName` 在知识库中查找记录。
3. **一致性校验**：比对记录中现有的 `architecture_id` 与请求中 `oldArchitectureId` 是否一致，不一致时返回警告信息。
4. **更新记录**：将对应记录的 `architecture_id` 更新为 `newArchitectureId`。
5. **返回结果**：组装并返回处理结果。

> 💡 **说明**：  
> - 本接口仅更新 DocSense 内部数据库中的映射关系，不触发重新解析任务。  
> - 若需在变更后重新解析，请单独调用 `/llm/weaponry` 接口。

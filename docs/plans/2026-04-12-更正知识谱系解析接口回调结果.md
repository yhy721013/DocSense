# 修复 weaponry 接口 source 字段返回文件开头而非对应 chunk 的问题

## 问题分析

`/llm/weaponry` 接口回调中 `analyseDataSource.source` 字段总是返回文件开头内容，而非 RAG 检索匹配的 chunk。

### 根因

通过诊断脚本对比发现，AnythingLLM 两个 API 返回的数据存在关键差异：

| API | text 内容 | 数量 |
|-----|----------|------|
| `chat` API 的 `sources` | 文档 `pageContent` 截断版（前约1037字符），**总是从文件开头** | 1个对象 |
| `vector-search` API | **真正的 chunk 文本**（100-150字符），各有独立 score | 6个对象 |

当前代码在 [_query_input_field](file:///c:/Files/Codes/DocSense/app/services/llm_service/weaponry_service.py#L152-L195) 中使用 `chat` API 返回的 `sources` 来构建 `analyseDataSource`，因此 `source` 字段永远是文件开头。

## 修复方案

在每次 RAG 查询后，额外调用 `vector-search` API 获取真实 chunk 数据来构建 `analyseDataSource`。

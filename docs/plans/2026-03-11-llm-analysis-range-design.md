# 文件解析范围约束设计

**日期：** 2026-03-11

**目标：** 为 `/llm/analysis` 增加“甲方范围优先、默认测试范围兜底”的约束能力，确保 `channel`、`country`、`format`、`maturity`、`architectureList` 的最终回调值只能来自请求范围或默认测试范围，超出范围时留空。

## 1. 背景与约束

- 最终联调时，范围以甲方请求为准。
- 当前测试期，甲方因数据敏感无法提供完整范围，因此后端需要提供默认模拟范围。
- 默认模拟范围只在请求缺失或为空时生效，不改变正式联调优先级。
- `architectureList` 默认结构不使用 `api-test.md` 中的示例节点，而是以 [rag_with_ocr.py](/e:/DocSense/rag_with_ocr.py) 中 `PROMPT` 展示的分类体系为准。
- 文件解析最终回调仍然遵循 [api-test.md](/e:/DocSense/api-test.md) 协议：
  - 命中范围内值则返回
  - 未命中或越界则留空
  - `architectureId` 未命中则返回 `0`

## 2. 正式行为定义

### 2.1 范围优先级

文件解析请求中以下字段视为“受约束范围”：

- `channel`
- `country`
- `format`
- `maturity`
- `architectureList`

处理顺序：

1. 若请求中该字段存在且为非空有效值，则以请求值为准。
2. 若请求中该字段缺失、为空数组或无有效元素，则注入默认测试范围。
3. 模型只允许在最终候选范围内选择值。
4. 服务端映射层对模型结果再次裁剪，确保越界值不能进入回调。

### 2.2 字段返回规则

- `country/channel/format/maturity`
  - 命中候选范围中的 `value` 时返回该 `value`
  - 未命中或越界时返回空字符串
- `architectureId`
  - 命中候选节点 `id` 时返回该数字
  - 未命中或越界时返回 `0`
- `fileDataItem` 的其他字段继续按当前抽取逻辑返回，不受这几个范围字段置空的影响

## 3. 默认测试范围

### 3.1 默认枚举

默认测试范围建议固定为：

- `country`
  - 美国
  - 俄罗斯
  - 日本
  - 英国
  - 法国
- `channel`
  - 装发
  - 军情
  - 科技
  - 训练
- `format`
  - 音频类
  - 文档类
  - 图片类
- `maturity`
  - 概念研究
  - 阶段成果
  - 定型成果

### 3.2 architectureList 默认结构

默认 `architectureList` 由后端固定生成，结构参考 [api-test.md](/e:/DocSense/api-test.md) 的元素字段，但内容以 [rag_with_ocr.py](/e:/DocSense/rag_with_ocr.py) 中现有分类体系为准：

- 军事基地
- 体系运用
- 装备型号
  - 空中装备
  - 水面装备
  - 水下装备
- 作战环境
- 作战指挥
  - 条令条例
  - 组织机构

实现时每个节点补齐：

- `id`
- `level`
- `name`
- `path`
- `pathName`
- `sort`

默认节点 ID 只用于测试期和范围约束，不与甲方正式库做主数据绑定。

## 4. 提示词约束

正式文件解析提示词需要调整为：

- 明确列出“最终候选范围”
- 明确要求只能输出候选范围内的值
- 明确要求：
  - `country/channel/format/maturity` 只能输出字符串 `value`
  - `architectureId` 只能输出数值 `id`
- 明确要求如果文档无法支撑判断则留空，而不是猜测或复述候选对象
- 明确禁止直接返回：
  - 候选对象
  - `key/value` 对象
  - 中文键名的范围对象

## 5. 服务端二次裁剪

后端映射层继续作为最终守门人：

- 即使模型返回了内容，也要再校验是否命中最终候选范围
- 若模型返回范围外值，后端强制置空
- 若模型返回对象结构，如 `{value: "美国", key: "02"}`，后端只提取其 `value` 再校验
- 对 `architectureId` 同理，只有候选范围内的 `id` 才允许保留

这样可以保证最终回调严格符合甲方范围约束，而不是单纯依赖模型自觉。

## 6. 实现边界

本次改动只覆盖文件解析正式链路，不扩大到报告链路：

- 纳入：
  - `/llm/analysis` 请求预处理
  - 文件解析正式提示词
  - 文件解析范围映射与裁剪
  - 文件解析测试样例与单测
- 不纳入：
  - `/llm/generate-report` 范围逻辑
  - 对话功能
  - 无关 UI 或调试页重构
  - 额外数据库结构改造

## 7. 验证策略

### 7.1 单元测试

验证：

- 请求显式传范围时，提示词与映射都以请求范围为准
- 请求缺失范围时，自动注入默认测试范围
- 模型返回范围内值时保留
- 模型返回范围外值时置空
- `architectureId` 只接受请求或默认范围内的节点 ID

### 7.2 手工联调

手工分两类：

- 显式传范围
  - 最终回调只能返回这些范围内的值
- 不传范围
  - 最终回调按默认测试范围约束

重点检查：

- `.runtime/mock_callback/last_callback.json`
- `/llm/check-task`
- `/llm/progress`
- AnythingLLM 原始返回与最终映射结果的差异

## 8. 风险与取舍

- 默认测试范围过大时，模型更容易“复述候选”而不是抽取，因此本次使用窄集合。
- `architectureList` 默认 ID 仅用于本地测试，正式联调仍必须以甲方请求中的真实 ID 为准。
- 当前变更只保证“范围约束”正确，不额外承诺提升所有 `fileDataItem` 字段的抽取质量。
